# 05 - Quick Watch Window

## Overview

This video introduces the Quick Watch window in the debugging environment. It explains how to monitor specific variables or expressions during a debugging session and how to keep them visible on the screen.

## Main Topics

### The Quick Watch Window

While the Autos window displays variables automatically, the Quick Watch window allows the developer to target and monitor a specific variable manually. This is useful when the focus is on a particular piece of data rather than all variables in the current scope.

### Opening and Using Quick Watch

To open this window, the program must be running in debugging mode. The developer highlights the specific variable to monitor and presses Shift + F9. This opens a dialog box displaying the variable's current value and memory address.

### Adding to the Watch List

To avoid reopening the Quick Watch dialog every time, the variable can be added to a persistent Watch list. By clicking "Add Watch" at the bottom of the Quick Watch window, the variable is pinned to the bottom of the screen for the duration of the debugging session.

### Monitoring Expressions and Functions

The Quick Watch window is not limited to simple variables. It can monitor different types of data, including arrays, functions (such as `MySum(a, b)` to observe the function execution), and custom mathematical expressions. Expressions can be evaluated and monitored using the "Reevaluate" option at the bottom of the screen.

## Key Takeaways

- The Quick Watch window is used to monitor specific variables or expressions manually.
- The shortcut to open the window is Shift + F9.
- Clicking "Add Watch" pins the variable to a persistent list at the bottom of the screen.
- It supports monitoring variables, arrays, functions, and custom expressions.
