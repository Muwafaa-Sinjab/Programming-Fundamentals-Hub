# 06 - Changing Values In Debugging Mode

## Overview

This video explains how to change the value of a variable during a debugging session. This modification is temporary and does not affect the original value written in the source code.

## Main Topics

### Temporary Value Modification

When the program is paused in debugging mode, the developer can change the stored value of any variable. This allows testing how the code behaves with different data without editing the actual program file. Once the debugging session ends, the variable returns to its original hardcoded value.

### Methods to Change Values

There are three places in the debugging interface where a variable's value can be modified:

- **Mouse Hover:** Placing the cursor over the variable in the code editor displays its value. Clicking on this value twice allows typing a new one.
- **Autos Window:** The value displayed in the Autos window can be double-clicked to replace it with a new temporary value.
- **Quick Watch Window:** The same double-click action can be performed on the variable's value inside the Quick Watch dialog.

## Key Takeaways

- Variable values can be changed temporarily during a debugging session.
- The original source code remains unchanged after the debugging session ends.
- A value is modified by double-clicking it and typing the new number.
- Changes can be applied directly in the code editor, the Autos window, or the Quick Watch window.
