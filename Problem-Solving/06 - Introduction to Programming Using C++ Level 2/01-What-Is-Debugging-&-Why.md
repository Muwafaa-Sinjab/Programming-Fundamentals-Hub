# 01 - What Is Debugging & Why

## Overview

This video introduces the concept of debugging in programming. It explains what debugging is, how debugging mode operates before generating the final executable file, and the three main types of errors that occur in programming.

## Main Topics

### What is Debugging

Debugging is the process of tracing code line by line to locate errors. During this process, it is possible to observe the values of variables and see exactly how data is stored in memory.

### Debugging Mode

Before creating the final executable file (File.exe), the code can be run in debugging mode. This mode allows the program to execute slowly, giving the developer the ability to track the code step by step rather than running it all at once. Using this mode significantly speeds up the process of finding and solving problems.

### Types of Programming Errors

There are three main types of errors in programming:

- **Syntax Error:** Mistakes in the writing rules of the code. This is the easiest type of error to find and fix, and it usually does not require debugging.
- **Logical Error:** A mistake in the logic or algorithm of the code. This is the most difficult type of error to detect because the code might run without crashing but produces wrong results. Debugging is primarily used to solve these errors.
- **Runtime Error:** An error that occurs while the program is running, which can cause the program to crash or behave unexpectedly.

## Key Takeaways

- Debugging allows line-by-line code tracing and memory inspection.
- Debugging mode executes the code before generating the executable file.
- Syntax errors are related to writing rules, while logical errors are related to the code's algorithm.
- Debugging is an essential tool for finding and fixing logical errors.
