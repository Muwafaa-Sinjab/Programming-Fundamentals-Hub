# 03 - More About Breakpoints

## Overview

This video covers additional techniques for working with breakpoints. It explains how to manage multiple breakpoints, navigate between them, move the execution point freely, and control the debugging session without removing the markers.

## Main Topics

### Working with Multiple Breakpoints

More than one breakpoint can be added to a program. When running in debugging mode, the program stops at the first breakpoint it reaches. Pressing the F5 shortcut moves the execution directly to the next available breakpoint.

### Moving the Execution Arrow

Besides stepping through code line by line, the yellow execution arrow can be grabbed with the mouse and dragged to any other line in the code. This allows the developer to skip lines or jump back to a previous section without executing the code in between.

### Stopping the Debugging Session

The debugging mode can be ended at any time by clicking the stop button located in the top toolbar. This stops the program execution completely and exits the debugging state.

### Disabling Breakpoints

A breakpoint can be disabled without deleting it from the code. When a breakpoint is disabled, the program runs fully from start to finish without pausing at that line. This is useful when a breakpoint is needed for later but is not required for the current test run.

## Key Takeaways

- Multiple breakpoints can be used to pause execution at different stages of the program.
- The F5 shortcut advances the program from one breakpoint to the next.
- The execution arrow can be dragged with the mouse to jump to any line manually.
- Debugging can be stopped using the top toolbar.
- Disabling a breakpoint keeps it in the code but allows the program to run without stopping.
