# 08 - IP Part 3 (IP Types)

## Overview

This video explains the different types of IP addresses. It covers the four main categories used to identify devices on local networks and the internet, and how each type functions differently depending on the need for reachability and permanence.

## Main Topics

### Public IP

A public IP address is a unique address that allows any device in the world to connect to a specific device. When a user subscribes to an internet service provider, the provider assigns a public IP address to their connection. This address exposes the device to the internet and makes it reachable from the outside.

### Private IP

A private IP address is used only within a local network, such as a home or office. Devices inside this network use private IPs to communicate with each other, but these addresses are not exposed to the public internet. Because private IPs are isolated to specific local networks, the same private IP address can exist in different networks without causing conflicts.

### Static IP

A static IP address is a fixed address that does not change over time. It is typically used for servers, security cameras, or businesses that require a permanent online address so users can always find them. Because the number of available static IPs is limited, they are more expensive and usually require a special request to the internet service provider.

### Dynamic IP

A dynamic IP address changes periodically, such as every time a device connects to the internet. This type is more common and cheaper because internet service providers share a pool of dynamic IPs among many users. In local networks, dynamic IPs are usually assigned automatically by the router, which prevents the manual errors that can happen when assigning IPs to a large number of devices by hand.

## Key Takeaways

- Public IP addresses allow devices to be reached from the global internet
- Private IP addresses are used only for communication within a local network
- Static IP addresses are fixed and are commonly used for servers and permanent services
- Dynamic IP addresses change periodically and are shared among users to save resources
- Using dynamic IPs in a local network helps avoid conflicts from manual assignment errors
