# 24 - FTP (File Transfer Protocol)

## Overview

This video introduces the File Transfer Protocol (FTP). It explains the specific purpose of this protocol, its main features for managing file transfers, and how it overcomes the limitations of standard web browsers when handling large files.

## Main Topics

### What is FTP?

FTP stands for File Transfer Protocol. It is a network protocol designed specifically for transferring files between a client and a server. While HTTP is the protocol used for viewing web pages and SMTP is used for sending emails, FTP is the dedicated protocol for uploading and downloading files.

### Key Features of FTP

FTP provides several capabilities that standard web browsers do not offer. It allows users to transfer multiple files at the same time. If an internet connection is lost during a transfer, FTP can resume the process from the exact point it stopped, rather than starting the download over from the beginning. Additionally, FTP allows users to manage files directly on the server, including the ability to rename, delete, or organize files into folders.

### Advanced Capabilities

FTP supports scheduling, which means users can set file transfers to start automatically at a specific time in the future. It also includes a queue system, allowing users to line up multiple files so they upload or download one after another. Furthermore, while web browsers typically restrict file downloads to a certain size limit, usually around 2 gigabytes, FTP does not have these restrictions. This makes it the preferred method for transferring very large files.

## Key Takeaways

- FTP stands for File Transfer Protocol and is dedicated entirely to file transfers
- It supports uploading and downloading multiple files simultaneously
- FTP can resume interrupted transfers from the exact point they stopped
- It allows direct file management on the server, such as renaming and deleting
- It has no standard file size limits and supports scheduled and queued transfers
