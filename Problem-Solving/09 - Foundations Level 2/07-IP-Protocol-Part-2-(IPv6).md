# 07 - IP Protocol Part 2 (IPv6)

## Overview

This video explains the limitations of IPv4 and introduces IPv6 as the solution. It covers the structure of an IPv6 address, its massive address capacity, and why both IPv4 and IPv6 currently coexist on the internet.

## Main Topics

### Why IPv6 Was Created

The 4.2 billion unique addresses provided by IPv4 are no longer sufficient to accommodate the massive number of internet-connected devices around the world. With the continuous growth in smartphones, computers, smart TVs, and other connected devices, a new generation of internet addresses was required. This led to the creation of IPv6.

### Structure of IPv6

An IPv6 address is 128 bits long, which equals 16 bytes. Instead of using standard decimal numbers like IPv4, IPv6 uses a hexadecimal format. The address is divided into 8 groups, with each group containing 4 hexadecimal characters separated by colons.

There are two ways to write an IPv6 address:
- **Long version**: All characters are written out completely (e.g., 2001:0db8:0000:0000:ff00:0000:0042:8330).
- **Short version**: Leading zeros within each group are removed to make the address shorter and easier to read (e.g., 2001:bd8:ff00:0:42:8330).

### Address Capacity

Because an IPv6 address is 128 bits long, the total number of unique addresses it can generate is approximately 340 trillion trillion trillion. This massive capacity ensures that the world will not run out of IP addresses for the foreseeable future.

### Coexistence of IPv4 and IPv6

When checking a device's network settings, both an IPv4 and an IPv6 address are often listed. This is because the transition to IPv6 is not yet complete globally. Some countries and networks have fully adopted IPv6, while others still rely heavily on IPv4. Because adoption is not universal, the internet currently runs on both protocols simultaneously. IPv4 will not be completely phased out until all global networks successfully adopt IPv6.

## Key Takeaways

- IPv6 was developed because the 4.2 billion addresses in IPv4 are no longer enough for global demand
- An IPv6 address is 128 bits long and uses a hexadecimal format
- IPv6 addresses can be written in a long version or a shortened version by removing leading zeros
- IPv6 provides approximately 340 trillion trillion trillion unique addresses
- Both IPv4 and IPv6 are currently used together because global adoption of IPv6 is not yet complete
